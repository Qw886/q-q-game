# Repository Guidelines

## 项目定位

`mahjong-link` 是使用 Cocos Creator 3.8.8 和 TypeScript 开发的 2D 麻将连连看游戏。长期目标按阶段推进：先完成可在浏览器运行的 Web Mobile 版本；Web 版本稳定后再适配微信小游戏。第一阶段只开发单机游戏，暂不开发微信登录、广告、支付、Spring Boot 后端、MySQL、Redis 和在线排行榜。

## 游戏规则

麻将必须紧密排列成一个完整的大长方形。玩家依次点击两张图案相同的麻将，若两张麻将之间存在最多两次转弯的连接路线，则可以消除。路线不能经过其他未消除麻将，但允许经过已经消除的位置，也允许从棋盘外围的虚拟空白区域绕行。

成功消除一对麻将后，本轮倒计时重新开始；倒计时归零时游戏失败。初始棋盘必须至少存在一条可以完全清空的解法。玩家选择错误消除顺序后可以形成死局；当麻将未清空但已找不到任何合法配对时，必须立即显示死局失败，不能让游戏一直卡住。

## 难度配置

所有模式配置必须集中放在配置文件中，不能把数字散落在不同代码文件里。

- 普通模式：8 列 x 10 行，80 张麻将，20 秒倒计时。
- 中等模式：10 列 x 12 行，120 张麻将，14 秒倒计时。
- 困难模式：12 列 x 16 行，192 张麻将，9 秒倒计时。

## 代码结构

业务代码主要放在 `assets/scripts/` 下，并按职责划分：

- `config`：难度和游戏配置。
- `core`：棋盘数据、寻路、生成器、合法移动搜索、死局检测和游戏状态。
- `view`：麻将节点、棋盘显示和连线显示。
- `ui`：主菜单、倒计时和结算界面。
- `storage`：本地最高分和设置。
- `bootstrap`：游戏启动入口。

纯算法代码尽量不要依赖 Cocos 的 `cc` 模块，方便独立测试。一个文件只负责一个主要功能。

## 编码要求

使用 TypeScript。使用清晰的类型，避免随意使用 `any`。类名、函数名和变量名使用清晰英文。关键算法添加中文注释，解释意图和边界条件。第一版使用文字和色块作为麻将占位素材，不复制其他游戏的源代码、图片、音乐和界面。

不擅自安装新的 npm 依赖。安装依赖前必须先说明用途并等待用户确认。

## 项目保护规则

不要修改或提交以下自动生成目录：`library/`、`temp/`、`build/`、`local/`。不要大范围手工修改 `scene`、`prefab` 和 `meta` 文件。

需要用户在 Cocos Creator 编辑器中手动执行的操作，写入 `docs/EDITOR_SETUP.md`，并写清楚点击哪个节点、添加哪个组件、设置什么属性。

## 开发流程

每次只完成一个阶段，不能一次性实现整个游戏。开始修改前必须阅读现有目录，说明准备修改哪些文件，并给出简短计划。

完成修改后必须检查 TypeScript 错误，运行当前能够运行的检查，总结创建和修改的文件，列出需要用户在 Cocos 编辑器中完成的操作，说明尚未解决的问题，然后停止，不要自动开始下一个阶段。

功能不能只创建空类或 TODO，必须有可以观察和验证的结果。遇到不确定的 Cocos API 时，先检查当前项目版本和已有类型定义，不要凭记忆编造。
